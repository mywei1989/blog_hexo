关于本blog

人们学会一项技能之后，总会想使用一下。
可惜在我身边，很多人都只是“想”，而没有去真正“做”过，或许有，只是我没真正发现而已。当然，过去的很多年，我也是这样的，但今天，就从上上周开始，就现在，我决定去做这件事，踏踏实实的把个人博客搭起来，然后再写起来。

ta们的想法

也许很多开发者会认为，个人博客是个简单到不屑于去投入的，做好这个体现不了自己的水平，没有“自己搭建的”博客，甚至没有博客，也不能说明那个开发者就一定水平不高。
过去的几年里，或者说本人正式从事IT行业开始，作为一名后端，虽然业务逻辑那一块用数据库可以清洗表达，但博客不是那种“内部管理系统”，脸面还是有点重要的，自己虽然能看懂一般的html+css+js，但水平确实有限，设计能力也差点，搭建博客这件事，还是算了吧，即使平时真有点冲动写点什么的的时候，也仅仅使用博客园或csdn,想自己折腾一下,就用wp搭建一个php的,什么主题都有，但真实懒啊，又不是搞php的，一个完美主义者的痛苦，很多人都不会懂。

后来又转了前端，而身边的同事，除了极个别能紧跟时代，但这些人中的大部分还是使用成熟的一体化的博客工具，GHOST，HEXO什么的。当然，写总比不写强，往大了说，也是知识的一种延续。
但是，现在我的是一名前(quan)端(zhan),像传统前端或后端的做法,真不是我的风格,或者说达不到”装逼”的程度。就像当理想慢慢被现实生活磨灭的时候，温水中的青蛙会有两种选择：意识到并跳出去了但同样也还是青蛙，生物学上能活多久就只能活多久；没意识到就被煮熟了，这也许也是命，因为活得长，不代表幸福，真按看动物的眼光看，如果在锅里死了的青蛙已经繁殖过了，而跳出去的孤独终生，这两只青蛙谁更成功？所以这个故事说明了一个道理：单身的人，别谈成功!!

talk is cheap,show you the code

本博客使用nodejs作为web容器,开发框架为express4+,数据库使用mongodb,模版引擎使用ejs。为什么使用这么个组合？jade不是比ejs更能装吗？很多东西真的不需要解释，这个要撕起来，我跟你撕几天几夜都没问题。源码在右侧最下方的那只github吉祥物里。
刚搭建起来,近期会将已有的其他博客网站本人的文章迁移过来,当然还会把本人一些懒得在网上发表的文章 笔记 草稿等也全部迁移过来。这个过程时间不确定，毕竟工作几年，各种小经验，小想法不少。
